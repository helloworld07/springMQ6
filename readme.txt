1.队列模式流程：
    a.基本监听模式(MessageListener)：消息监听器、消息目的地和连接工厂会被注册到消息监听容器中，这时，就已经有消费者等待在队列中，
        当消息通过消息生产者（通过Spring提供的JMS工具类jmsTemplate）发送给目的地时，就会被监听此目的地的
        消息监听器发现，从而触发监听器操作，接收消息。这就是消息从发出到activemq再从队列被获取接收的整个过程。
    b.有回复的监听器(SessionAwareMessageListener):原理类似，接收消息的同时可以回复消息给目的地。本实例中，回复消息给queue队列，
        此时，sessionAwareQueue用来接收和发送消息，queueDestination只用来接收监听器发来的回复消息。
    c.同时实现了上面两种接口的监听器(MessageListenerAdapter)：可以进行消息转换，需在监听器里配置转换器。可以发送实体类的消息（非String）,
        监听器通过转换后得到实体并可以通过实体的toString方法输出实例内容。
2.订阅者模式：
    基本原理与队列差不多，不同之处在于订阅者（消费者）会随监听器一开始就被注册到ActiveMQ中，即发布者（生产者）为发布消息时订阅者就需要就位，
    待发布者发布消息后，订阅者会第一时间调用方法处理消息。
